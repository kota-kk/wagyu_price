# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
 - リモートリポジトリ  
→Web上に配置するリポジトリ。チームメンバーで共有する資産なので直接触りたくない。

 - ローカルリポジトリ  
→自身のPC(ローカル環境)で管理しており、リモートリポジトリで取り込むまで直接影響させずに修正を行うことが可能になる


## プッシュとマージの違いは何でしょうか？
 - プッシュ  
→コミットした差分の挿入を行う（基本的にローカル→リモートで行う）

 - マージ  
→ローカル同士、もしくはリモート同士で使用する。  
例）プッシュした差分の取り込みを別ブランチで行う


## コミットとプッシュの違い
 - コミット  
→修正差分をローカルリポジトリに保存する

 - プッシュ  
→ローカルリポジトリに保存した修正差分をリモートリポジトリに反映させる（差分履歴を追加する）


## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
 - コミット単位でコードの意図がわかるよう、どのような修正を行なったのかを明確に記載する


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
 - 承認者の確認を挟むかどうかの違い


## コンフリクトを起こしてしまった場合、どう対処すべきですか？
 - 上長に共有のうえ、競合を起こしている相手と取り込み内容の確認を行う。
